---
title: Спецификации извлечения
description: Как создаются правила поиска
---

Внутренние классы спецификаций никогда не создаются напрямую. Используйте фабрику `Extract`:

```python
Extract.code(...)   # коды подтверждения
Extract.link(...)   # ссылки http/https
Extract.regex(...)  # произвольный regex
```

Каждый вызов возвращает неизменяемый объект-спецификацию, который **проходит
валидацию в момент создания** — некорректные настройки вызывают исключение
немедленно (например, `InvalidCodeSpecError`), ещё до запуска поиска.

Важные значения по умолчанию:

- `Extract.code()` находит токены только из цифр длиной ≥ 1, с границами по
  классу символов (токен должен граничить с символами вне разрешённого набора).
- `Extract.link()` находит каждую ссылку `http`/`https` в тексте; добавьте
  фильтры, чтобы сузить выборку.
- `Extract.regex(pattern)` компилирует `pattern` сразу; некорректный шаблон
  вызывает `RegexExtractError` здесь, а не во время поиска.

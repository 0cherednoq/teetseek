## Числовой код

```python
TextSeek(text).extract_one(Extract.code(length=6, digits=True))
```

## Буквенно-цифровой код

```python
Extract.code(length=8, digits=True, alphabet=True)  # matches A1B2C3D4
```

## Код с символами

```python
Extract.code(min_length=8, max_length=16, digits=True, alphabet=True, symbols=True)
# matches A8F3-K2P9
```

Стандартный набор символов — `-` и `_`.

## Требование каждого типа

```python
Extract.code(length=8, digits=True, alphabet=True, require_each_type=True)
# matches A1B2C3D4, rejects 12345678 and ABCDEFGH
```

## Рядом с фразой

```python
from textseek import TextSeek, Extract

text = "Ignore 111111. Your verification code is 654321."
code = TextSeek(text).extract_one(
    Extract.code(length=6, digits=True, near="verification code")
)
print(code.value)  # 654321
```

`window` символов сканируется с **каждой стороны** фразы (300 по умолчанию). Коды,
которые появляются **после** фразы, важнее кодов перед ней (обычный паттерн «ваш
код — XXX»), и ближний важнее дальнего. Если после фразы кода нет, берётся
ближайший перед ней:

```python
TextSeek("G-447103 is your verification code").extract_one(
    Extract.code(length=6, digits=True, near="verification code")
).value  # 447103
```

При использовании `near` метод `extract_all` возвращает совпадения по рангу
(сначала лучшее), поэтому для первого примера это `["654321", "111111"]`.

## Строгие границы

Код сопоставляется как целый токен, ограниченный символами **вне разрешённого
класса символов**. `Extract.code(length=6, digits=True)` не совпадает с `123456`
внутри `1234567890` (все соседи — цифры). Но `Extract.code(length=4,
digits=True)` совпадает с `1234` внутри `ab1234`, потому что буквы не входят в
класс цифр.
